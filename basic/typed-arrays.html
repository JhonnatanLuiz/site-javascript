JavaScript Typed Arrays

Typed Arrays foram projetados para manipular dados binários de forma eficiente. Diferente de arrays regulares, typed arrays são buffers de tamanho fixo que armazenam elementos de tipos fixos, como inteiros de 8 bits ou números de 32 bits.

Benefícios dos Typed Arrays

Typed Arrays fornecem uma forma eficiente de manipular dados binários. JavaScript pode passá-los diretamente para qualquer função sem converter os dados para outra representação.

🚀 Typed Arrays são ideais para:

  • WebGL e Canvas: Renderização gráfica rápida e processamento de imagens
  • File APIs: Leitura e escrita rápida de arquivos locais
  • Media APIs: Manipulação rápida de dados de áudio e vídeo
  • WebSockets: Transferência eficiente de dados binários pela rede

Diferenças dos Arrays Regulares

  • ✓ Tamanho Fixo: Typed Arrays não podem ser redimensionados dinamicamente com métodos como push() ou pop().
  • ✓ Restrição de Tipo: Elementos devem seguir o tipo de dados especificado do typed array.
  • ✓ Buffer Subjacente: Typed Arrays são visualizações de um ArrayBuffer, permitindo manipulação direta de dados binários.

Criando Typed Arrays

Exemplo 1: Criar a partir do tamanho

const myArr = new Uint8Array(5);
// Cria um array tipado de 5 bytes

Exemplo 2: Criar a partir de um array

const myArr = new Uint8Array([0, 1, 2, 3, 4]);

Exemplo 3: Usando o método of()

const myArr = Uint8Array.of(0, 1, 2, 3, 4);

Exemplo 4: Usando o método from()

const myArr = Uint8Array.from([0, 1, 2, 3, 4]);

Inteiros de 8 Bits

Exemplo 5: Int8Array e Uint8Array

// Inteiros de 8 bits com sinal (-128 a 127)
const signedArr = new Int8Array(10);

// Inteiros de 8 bits sem sinal (0 a 255)
const unsignedArr = new Uint8Array(10);

// Uint8ClampedArray - valores são "clampados" entre 0 e 255
const clampedArr = new Uint8ClampedArray(10);

Uint8Array vs Uint8ClampedArray

⚠️ A diferença entre Uint8Array e Uint8ClampedArray é como os valores são adicionados:

  • Uint8ClampedArray: Valores fora do intervalo 0-255 são "clampados" para 0 ou 255
  • Uint8Array: Pega apenas os primeiros 8 bits do valor

Exemplo 6: Diferença entre Uint8Array e Uint8ClampedArray

const uint8 = new Uint8Array(1);
const clamped = new Uint8ClampedArray(1);

uint8[0] = 300;     // Resultado: 44 (300 & 255)
clamped[0] = 300;  // Resultado: 255 (clampado)

Inteiros de 16 e 32 Bits

Exemplo 7: Int16Array, Uint16Array, Int32Array, Uint32Array

// 16-bit - 2 bytes por elemento
const int16 = new Int16Array([-32768, 0, 32767]);
const uint16 = new Uint16Array([0, 32768, 65535]);

// 32-bit - 4 bytes por elemento
const int32 = new Int32Array([-2147483648, 2147483647]);
const uint32 = new Uint32Array([0, 4294967295]);

Números de Ponto Flutuante

Exemplo 8: Float32Array e Float64Array

// Float32Array - 4 bytes, precisão normal
const float32 = new Float32Array([3.14, 2.718, 1.618]);

// Float64Array - 8 bytes, precisão dupla
const float64 = new Float64Array([3.141592653589793]);

📌 Nota Importante

Typed Arrays não são arrays comuns.

  • Array.isArray() retorna false para typed arrays
  • Muitos métodos de array (como push e pop) não são suportados
