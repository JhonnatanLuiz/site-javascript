JavaScript Comments

Comentários JavaScript podem ser usados para explicar o código JavaScript e torná-lo mais legível.

Comentários JavaScript também podem ser usados para prevenir execução, ao testar código alternativo.

Comentários de Linha Única

Comentários de linha única começam com //.

Qualquer texto entre // e o final da linha será ignorado pelo JavaScript (não será executado).

Exemplo 1 - Comentários Antes do Código

Este exemplo usa comentários de linha única antes de cada linha de código:

// Muda o heading:
document.getElementById("myH").innerHTML = "Minha Primeira Página";

// Muda o parágrafo:
document.getElementById("myP").innerHTML = "Meu primeiro parágrafo.";

Exemplo 2 - Comentários ao Final da Linha

Este exemplo usa comentários no final de cada linha para explicar o código:

let x = 5;      // Declara x, atribui o valor 5
let y = x + 2;  // Declara y, atribui o valor de x + 2

Comentários Multilinha

Comentários multilinha começam com /* e terminam com */.

Qualquer texto entre /* e */ será ignorado pelo JavaScript.

Exemplo - Bloco de Comentários

Este exemplo usa um comentário multilinha (bloco de comentário) para explicar o código:

/*
O código abaixo irá mudar
o heading com id = "myH"
e o parágrafo com id = "myP"
na minha página web:
*/
document.getElementById("myH").innerHTML = "Minha Primeira Página";
document.getElementById("myP").innerHTML = "Meu primeiro parágrafo.";

💡 Quando Usar Cada Tipo:

  • • É mais comum usar comentários de linha única (//)
  • • Blocos de comentários (/* */) são frequentemente usados para documentação formal

Comparação de Tipos de Comentários

Tipo Sintaxe Uso Recomendado
Linha Única // comentário Explicações rápidas, notas curtas
Multilinha /* comentário */ Documentação formal, explicações longas

Usando Comentários para Prevenir Execução

Usar comentários para prevenir execução de código é adequado para testes de código.

Adicionar // na frente de uma linha de código transforma a linha de executável para comentário.

Exemplo - Desabilitando Uma Linha

Este exemplo usa // para prevenir a execução de uma das linhas de código:

//document.getElementById("myH").innerHTML = "Minha Primeira Página";
document.getElementById("myP").innerHTML = "Meu primeiro parágrafo.";

Exemplo - Desabilitando Múltiplas Linhas

Este exemplo usa um bloco de comentário para prevenir a execução de múltiplas linhas:

/*
document.getElementById("myH").innerHTML = "Minha Primeira Página";
document.getElementById("myP").innerHTML = "Meu primeiro parágrafo.";
*/

document.getElementById("result").innerHTML = "Código comentado não foi executado!";

🧪 Dicas de Teste e Debug:

  • ✓ Use // para desabilitar rapidamente uma linha durante testes
  • ✓ Use /* */ para desabilitar blocos grandes de código
  • ✓ Útil para comparar diferentes implementações
  • ✓ Facilita o debug ao isolar problemas

Boas Práticas com Comentários

✅ FAÇA

  • • Explique o porquê, não o o quê
  • • Mantenha comentários atualizados
  • • Use comentários para TODOs e FIXMEs
  • • Documente funções complexas
  • • Seja claro e conciso

❌ NÃO FAÇA

  • • Não comente código óbvio
  • • Não deixe código comentado no produto final
  • • Não use comentários redundantes
  • • Não escreva comentários confusos
  • • Não ignore comentários desatualizados

Exemplos de Bons Comentários

// BOM: Explica o porquê
// Usamos setTimeout para evitar bloqueio da UI
setTimeout(processData, 0);

// RUIM: Explica o óbvio
// Incrementa x em 1
x++;

// BOM: Documenta função complexa
/**
 * Calcula o fatorial de um número
 * @param {number} n - O número para calcular
 * @returns {number} O fatorial de n
 */
function factorial(n) {
  return n <= 1 ? 1 : n * factorial(n - 1);
}
