JavaScript Let

A palavra-chave let foi introduzida em ES6 (2015).

✅ Características do let:

  • • Variáveis declaradas com let têm Escopo de Bloco (Block Scope)
  • • Variáveis declaradas com let devem ser Declaradas antes do uso
  • • Variáveis declaradas com let não podem ser Redeclaradas no mesmo escopo

Escopo de Bloco (Block Scope)

Antes do ES6 (2015), JavaScript não tinha Escopo de Bloco.

JavaScript tinha apenas Escopo Global e Escopo de Função.

O ES6 introduziu as duas novas palavras-chave JavaScript: let e const.

Essas duas palavras-chave forneceram Escopo de Bloco em JavaScript.

Variáveis declaradas com let

Variáveis declaradas dentro de um bloco { } não podem ser acessadas de fora do bloco:

{
  let x = 2;
}
// x NÃO pode ser usado aqui

Escopo Global

Variáveis declaradas com var sempre têm Escopo Global.

Variáveis declaradas com a palavra-chave var não podem ter escopo de bloco:

Variáveis declaradas com var

Variáveis declaradas com var dentro de um bloco { } podem ser acessadas de fora do bloco:

{
  var x = 2;
}
// x PODE ser usado aqui

⚠️ Problema: Com var, você perde o controle do escopo da variável!

Não Pode Ser Redeclarada

Variáveis definidas com let não podem ser redeclaradas.

Você não pode acidentalmente redeclarar uma variável declarada com let.

❌ Com let você NÃO pode fazer isso:

let x = "John Doe";
let x = 0;

// SyntaxError: 'x' has already been declared

✅ Com var você PODE fazer isso:

var x = "John Doe";
var x = 0;

// Funciona, mas é uma má prática!

Redeclarando Variáveis

Redeclarar uma variável usando a palavra-chave var pode impor problemas.

Redeclarar uma variável dentro de um bloco também redeclara a variável fora do bloco:

❌ Problema com var

var x = 10;
// Aqui x é 10

{
  var x = 2;
  // Aqui x é 2
}

// Aqui x é 2 (VAZOU do bloco!)

✅ Solução com let

Redeclarar uma variável usando a palavra-chave let pode resolver este problema.

Redeclarar uma variável dentro de um bloco não redeclara a variável fora do bloco:

let x = 10;
// Aqui x é 10

{
  let x = 2;
  // Aqui x é 2
}

// Aqui x é 10 (Mantém o valor original!)

Diferença Entre var, let e const

Escopo Redeclarar Reatribuir Hoisted Vincula this
var ❌ Não ✅ Sim ✅ Sim ✅ Sim ✅ Sim
let ✅ Sim ❌ Não ✅ Sim ❌ Não ❌ Não
const ✅ Sim ❌ Não ❌ Não ❌ Não ❌ Não

✅ O que é Bom?

  • • let e const têm escopo de bloco
  • • let e const não podem ser redeclaradas
  • • let e const devem ser declaradas antes do uso
  • • let e const não vinculam ao this
  • • let e const não sofrem hoisting

❌ O que Não é Bom?

  • • var não precisa ser declarada
  • • var sofre hoisting
  • • var vincula ao this

Regras de Redeclaração

Redeclarando com var é permitido

var x = 2;     // Permitido
var x = 3;     // Permitido

❌ Com let, redeclarar no mesmo bloco NÃO é permitido

var x = 2;     // Permitido
let x = 3;     // NÃO permitido

{
  let x = 2;   // Permitido
  let x = 3;   // NÃO permitido
}

{
  let x = 2;   // Permitido
  var x = 3;   // NÃO permitido
}

✅ Redeclarar com let em outro bloco É permitido

let x = 2;   // Permitido

{
  let x = 3;   // Permitido (diferente escopo)
}

{
  let x = 4;   // Permitido (diferente escopo)
}

Hoisting com Let

✅ Hoisting com var

Variáveis definidas com var sofrem hoisting para o topo e podem ser inicializadas a qualquer momento.

Significado: Você pode usar a variável antes de declará-la:

carName = "Volvo";
var carName;

// Funciona! (mas é má prática)

❌ Hoisting com let

Variáveis definidas com let também sofrem hoisting para o topo do bloco, mas não são inicializadas.

Significado: Usar uma variável let antes de declará-la resultará em ReferenceError:

carName = "Saab";
let carName = "Volvo";

// ReferenceError!

🔍 Zona Morta Temporal (Temporal Dead Zone)

O período entre o início do bloco e a declaração da variável let é chamado de Temporal Dead Zone. Durante este período, a variável existe mas não pode ser acessada.

Suporte dos Navegadores

As palavras-chave let e const não são suportadas no Internet Explorer 11 ou anteriores.

🎯 Quiz Rápido

Pergunta: Qual é o uso correto da palavra-chave let?

📝 Resumo

  • • let tem escopo de bloco, var não tem
  • • let não pode ser redeclarada no mesmo escopo
  • • let deve ser declarada antes do uso (Temporal Dead Zone)
  • • let pode ser redeclarada em blocos diferentes
  • • Use let sempre que precisar de uma variável que pode ser reatribuída
