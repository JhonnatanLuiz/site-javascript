JavaScript Const

A palavra-chave const foi introduzida em ES6 (2015).

🔒 Características do const:

  • • Variáveis definidas com const não podem ser Redeclaradas
  • • Variáveis definidas com const não podem ser Reatribuídas
  • • Variáveis definidas com const têm Escopo de Bloco

Não Pode Ser Reatribuída

Uma variável definida com a palavra-chave const não pode ser reatribuída:

❌ Exemplo - Tentando Reatribuir const

const PI = 3.141592653589793;
PI = 3.14;      // Isso dará um erro
PI = PI + 10;   // Isso também dará um erro

Deve Ser Atribuída

Variáveis JavaScript const devem receber um valor quando são declaradas:

✅ Correto

const PI = 3.14159265359;

❌ Incorreto

const PI;
PI = 3.14159265359;

// SyntaxError!

Quando Usar JavaScript const?

Sempre declare uma variável com const quando você souber que o valor não deve ser alterado.

📌 Use const quando você declarar:

  • • Um novo Array
  • • Um novo Object
  • • Uma nova Function
  • • Um novo RegExp

Objetos e Arrays Constantes

A palavra-chave const é um pouco enganosa.

Ela não define um valor constante. Ela define uma referência constante para um valor.

❌ Por causa disso você NÃO PODE:

  • ✗ Reatribuir um valor constante
  • ✗ Reatribuir um array constante
  • ✗ Reatribuir um objeto constante

✅ Mas você PODE:

  • ✓ Alterar os elementos de um array constante
  • ✓ Alterar as propriedades de um objeto constante

Arrays Constantes

✅ Você PODE alterar os elementos de um array constante

// Você pode criar um array constante:
const cars = ["Saab", "Volvo", "BMW"];

// Você pode alterar um elemento:
cars[0] = "Toyota";

// Você pode adicionar um elemento:
cars.push("Audi");

❌ Mas você NÃO PODE reatribuir o array

const cars = ["Saab", "Volvo", "BMW"];

cars = ["Toyota", "Volvo", "Audi"];    // ERRO

Objetos Constantes

✅ Você PODE alterar as propriedades de um objeto constante

// Você pode criar um objeto const:
const car = {type:"Fiat", model:"500", color:"white"};

// Você pode alterar uma propriedade:
car.color = "red";

// Você pode adicionar uma propriedade:
car.owner = "Johnson";

❌ Mas você NÃO PODE reatribuir o objeto

const car = {type:"Fiat", model:"500", color:"white"};

car = {type:"Volvo", model:"EX60", color:"red"};    // ERRO

Escopo de Bloco

Declarar uma variável com const é semelhante a let quando se trata de Escopo de Bloco.

O x declarado no bloco, neste exemplo, não é o mesmo que o x declarado fora do bloco:

Exemplo - Block Scope com const

const x = 10;
// Aqui x é 10

{
  const x = 2;
  // Aqui x é 2
}

// Aqui x é 10

Diferença Entre var, let e const

Escopo Redeclarar Reatribuir Hoisted Vincula this
var ❌ Não ✅ Sim ✅ Sim ✅ Sim ✅ Sim
let ✅ Sim ❌ Não ✅ Sim ❌ Não ❌ Não
const ✅ Sim ❌ Não ❌ Não ❌ Não ❌ Não

✅ O que é Bom?

  • • let e const têm escopo de bloco
  • • let e const não podem ser redeclaradas
  • • let e const devem ser declaradas antes do uso
  • • let e const não vinculam ao this
  • • let e const não sofrem hoisting

❌ O que Não é Bom?

  • • var não precisa ser declarada
  • • var sofre hoisting
  • • var vincula ao this

Regras de Redeclaração

Redeclarando com var é permitido em qualquer lugar

var x = 2;     // Permitido
var x = 3;     // Permitido
x = 4;         // Permitido

❌ Redeclarar var ou let existente para const NÃO é permitido

var x = 2;     // Permitido
const x = 2;   // NÃO permitido

{
  let x = 2;     // Permitido
  const x = 2;   // NÃO permitido
}

{
  const x = 2;   // Permitido
  const x = 2;   // NÃO permitido
}

❌ Reatribuir const existente NÃO é permitido

const x = 2;     // Permitido
x = 2;           // NÃO permitido
var x = 2;       // NÃO permitido
let x = 2;       // NÃO permitido
const x = 2;     // NÃO permitido

{
  const x = 2;   // Permitido
  x = 2;         // NÃO permitido
  var x = 2;     // NÃO permitido
  let x = 2;     // NÃO permitido
  const x = 2;   // NÃO permitido
}

✅ Redeclarar const em outro escopo/bloco É permitido

const x = 2;       // Permitido

{
  const x = 3;   // Permitido
}

{
  const x = 4;   // Permitido
}

Hoisting com Const

✅ Hoisting com var

Variáveis definidas com var sofrem hoisting para o topo e podem ser inicializadas a qualquer momento.

carName = "Volvo";
var carName;

// Funciona!

❌ Hoisting com const

Variáveis definidas com const também sofrem hoisting, mas não são inicializadas.

alert(carName);
const carName = "Volvo";

// ReferenceError!

🎯 Quiz Rápido

Verdadeiro ou Falso: Variáveis criadas com a palavra-chave const nunca podem mudar seu valor.

📝 Resumo

  • • const define uma referência constante, não um valor constante
  • • Variáveis const devem ser atribuídas quando declaradas
  • • const não pode ser redeclarada ou reatribuída no mesmo escopo
  • • Você PODE alterar elementos de arrays e propriedades de objetos const
  • • Use const para arrays, objetos, funções e valores que não devem ser reatribuídos
